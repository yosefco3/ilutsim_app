# 03 · עמדות עם דרישות (Positions)

**מטרה:** "לב המערכת". כל שורה בסידור = עמדה = דרישה למאבטח אחד. מסך שבו מגדירים **פעם אחת**
את כל ~30 העמדות (ע.אחמ"ש, חש"י, ארנונה, קומה 6, סייר-1, ב-4, מבקרת, רכב סיור...) — מתוך התמונה האמיתית.

**מה נבנה:**
- מודל `Position` (שייך ל-`ActivationProfile`) + מיגרציה — שם, משמרת (MORNING/AFTERNOON/NIGHT),
  ימים פעילים, **שעות פר-יום**, מאפיינים נדרשים.
- אוצר-דרישות **גמיש/קונפיגורבילי** (אחמ"ש / חמוש / רכב עירייה / רוני / הליכה מרובה...) —
  כי אפיון המאבטחים עוד לא נסגר; לא לקודד נוקשה.
- API CRUD + מסך הגדרת עמדה.

**קריטריון הצלחה (המשתמש מריץ):** מוסיף עמדה אמיתית (למשל "ארנונה", א'-ה', שעות פר-יום, דרישות) →
עורך → מוחק → מרענן → נשמר נכון. מוסיף כמה עמדות מכל משמרת ורואה את כולן ברשימה.

**קבצים צפויים:** `models/position.py`, מיגרציה, repo/service/controller, schemas,
`frontend/admin/src/pages/PositionsPage.jsx`.
