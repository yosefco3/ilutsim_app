# 01 · אקסל + פייפליין ייבוא אילוצים

**מטרה כוללת:** להעלות אקסל אילוצים (בפורמט `דוגמה_אילוצים_מאבטחים.xlsx`), **לראות את כל הנתונים
יוצאים נקיים ומסודרים — כולל הערות, חלונות ממוזגים ושעות נכונות** — ואז לשמור אותם ל**מודל הזמינות הקיים**
כך שבונה הסידור (משימות 04+) קורא ממקור אמת אחד.

> ⚠️ לפני שמתחילים: קרא את **`STAGE_B_PROMPTS/README.md` → "הכללים הנעולים"**. הוא מקור האמת
> לכל הסמנטיקה (זמין/לא-זמין/ריק, en-dash, כלל האיחוד 12≠13.5, מיפוי ערב→`AFTERNOON`, התאמה לפי שם,
> שבוע מהכותרת, כתיבה ל-`WeeklySubmission`/`DailyStatus`/`ShiftWindow`). אין לסטות ממנו בלי אישור.

**קריטריון הצלחה כולל (המשתמש מריץ):** נכנס למסך ייבוא באדמין → מעלה את `דוגמה_אילוצים_מאבטחים.xlsx` →
רואה טבלה נקייה: 5 מאבטחים, חלונות מאוחדים עם שעות נכונות (דנה ראשון = 07:00–23:00; מי שיש לו חפיפה →
שעות לפי האיחוד), הערות, ואיזה שם זוהה/לא זוהה → לוחץ "אשר ייבוא" → נכנס למסך **"הגשות"** הקיים לשבוע →
רואה את כל המאבטחים והזמינות שלהם.

## מקרי-קצה שחייבים מענה (checklist)

- [ ] 3 שורות לכל מאבטח → רשומה אחת (שם/טלפון/הערות רק בשורה הראשונה).
- [ ] מיפוי משמרת: בוקר→`MORNING`, ערב→`AFTERNOON`, לילה→`NIGHT`.
- [ ] טווח שעות עם en-dash (`–`, U+2013) **וגם** מקף רגיל (`-`).
- [ ] `זמין` בלי שעות = כל המשמרת באותו יום.
- [ ] `לא זמין` **ותא ריק** = אין זמינות (שקולים ברמת התא).
- [ ] לילה חוצה חצות (`23:00–07:00`).
- [ ] חפיפות → איחוד; שעות מהאיחוד (12 לא 13.5).
- [ ] טלפון בפורמטים שונים (לא קריטי — הזהות לפי שם).
- [ ] זהות = שם: מצאי-או-צור מאבטח לפי שם (גם רשומה שנכתבה ידנית נכנסת). **בלי דחייה, בלי שמות כפולים.**
- [ ] שבוע היעד מטווח התאריכים בכותרת.
- [ ] ולידציה: שעה לא תקינה / מאבטח כפול / שורת-משמרת חסרה → מדווח, לא קורס.
