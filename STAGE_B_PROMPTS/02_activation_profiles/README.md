# 02 · פרופילי הפעלה (Activation Profiles)

**מטרה:** הישות הבסיסית שמכילה עמדות — "מצב" שבו עובדים (שגרה / חג / אירוע). פרופיל = אוסף עמדות.
זו הקופסה שכל השאר נתלה עליה, ולכן נבנית לפני העמדות.

**מה נבנה:**
- מודל `ActivationProfile` (+ מיגרציה Alembic) — שם, סוג (שגרה/חג/אירוע), תיאור.
- API: יצירה / רשימה / שכפול / עריכת-שם / מחיקה (`/admin/profiles`, מאחורי `require_admin_role`).
- מסך ניהול פרופילים פשוט (frontend admin).
- זריעת פרופיל דיפולטי **"שגרה"**.

**קריטריון הצלחה (המשתמש מריץ):** נכנס למסך → רואה "שגרה" → יוצר "חג" → משכפל ל"חג שני" →
משנה שם → מוחק → מרענן את הדף → הכל נשמר ומשתקף נכון.

**הערה:** בסוף המשימה הפרופיל **ריק מעמדות** — זו הנחת-היסוד; העמדות נכנסות במשימה 03.
שיוך פרופיל↔שבוע יגיע עם הלוח (משימה 04).

**קבצים צפויים:** `models/activation_profile.py`, מיגרציה, `repositories/`, `services/`,
`controllers/profile_controller.py`, `schemas/`, `frontend/admin/src/pages/ProfilesPage.jsx`.

**תלוי ב:** —

> פרומפטים מלאים ייכתבו כשנגיע למשימה זו.
